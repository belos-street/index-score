# Index Score — 大盘指数量化打分 Agent

> 基于多因子分位数模型的大盘指数估值评估系统，融合 Agent 实现智能化投资解读。

Index Score 是一款面向长期价值投资者的量化工具，通过 **模板化多因子加权模型** 对主流大盘指数进行 1-9 分量化打分。系统从真实市场数据源实时拉取 PE、PB、股息率等核心估值指标，结合历史分位数统计和价格位置分析，将复杂的估值分析过程浓缩为一个直观的评分数字。

分数越低，代表指数当前估值越便宜，长期投资价值越高。

```
 1 分 ─── 极便宜    买入信号强烈
 3 分 ─── 便宜      具备安全边际
 5 分 ─── 中性      估值合理区间
 7 分 ─── 偏贵      需谨慎观望
 9 分 ─── 极贵      估值泡沫风险
```

---

## 核心特性

### 多因子分位数评分引擎

摒弃绝对估值的局限性，采用 **5 年滚动窗口分位数** 作为评分基准。PE 绝对值在不同市场环境下含义截然不同（牛市 15 倍 PE 可能偏贵，熊市 15 倍 PE 可能便宜），而分位数回答的是"当前估值在历史中处于什么位置"这一核心问题。

**技术要点：**

- **因子分解**：将指数估值拆解为 PE 分位、PB 分位、股息率分位、价格位置分位四个独立因子
- **模板化加权**：不同类型指数使用不同权重模板（红利型重股息率、价值型重 PE/PB、成长型重价格动量）
- **缺失因子权重重分配**：当某因子数据不可用时，其权重按比例自动分配至其余因子，保证评分链路不中断
- **5 档离散打分**（1/3/5/7/9）：跳过偶数档位，强制每个区间具备明确的买入/观望/卖出含义，消除模棱两可

### Agent 智能解读

集成 Agent 框架，将量化打分结果转化为专业投资解读。Agent 内置 `get_index_score`、`compare_indexes`、`generate_report` 等工具，能够基于打分明细自主推理，生成包含估值分析、因子拆解、投资建议的自然语言报告。

### 双源并行容灾架构

```
估值数据：理杏仁 Open API
   │
   ├── 成功 → PE/PB/股息率 + 5年分位点（一次返回）
   │
   └── 失败 → 3 次重试（间隔 3s）
                │
                ├── 成功 → 清洗 → 打分
                │
                └── 失败 → 降级（缺失因子权重自动重分配）

行情数据：AkShare
   │
   ├── 成功 → 历史价格 → 计算价格位置
   │
   └── 失败 → 3 次重试（间隔 3s）
                │
                └── 失败 → FetchError
```

数据层采用双源并行策略：**估值数据走理杏仁 API**（覆盖所有 A 股指数，PE/PB/股息率 + 分位点一次返回），**行情数据走 AkShare**（历史价格稳定可靠，用于计算价格位置）。估值数据拉取失败时不会阻断流程——系统会以缺失因子的方式继续完成打分，并通过权重自动重分配保证结果的合理性。

## 打分模型

### 模板化因子权重

系统不使用"一套公式打天下"的粗暴方式，而是根据指数类型特征定制专属打分模板：

| 模板 | 适用指数 | 股息率 | PE | PB | 价格位置 |
|------|---------|:------:|:--:|:--:|:-------:|
| **红利型** | 中证红利、中证红利低波、标普红利低波 | **40%** | 35% | — | 25% |
| **价值型** | 国证价值100、自由现金流 | 30% | **40%** | **30%** | — |
| **成长型** | 纳指 | 15% | **50%** | — | 35% |
| **宽基型** | 标普500 | 35% | 35% | — | 30% |

**设计哲学：**

- **红利型**：股息率权重最高（40%）。高股息率 = 低价格 + 高分红 + 现金流充足，是最难被会计操纵的"便宜信号"
- **价值型**：引入 PB 作为第二估值维度。低 PB 对资产密集型行业（金融、能源）尤其有效，与 PE 形成双估值验证
- **成长型**：PE 独占 50% 权重。科技公司估值由盈利预期驱动，价格动量捕捉中期趋势，股息率几乎无意义（科技公司偏好回购）
- **宽基型**：三因子均衡分配，不做激进倾斜。宽基指数行业分散，避免单一因子主导

### 分位数计算

```
分位数 = 当前值在近 N 年历史数据中的百分位排名

PE 分位（5 年）：当前 PE 在过去 5 年日频 PE 数据中的排位
股息率分位（5 年）：当前股息率在过去 5 年日频数据中的排位
价格位置（3 年）：(当前价 - 3 年最低) / (3 年最高 - 3 年最低)
```

### 分位 → 分数映射

| 分位区间 | 分数 | 估值标签 | 投资含义 |
|:-------:|:----:|:-------:|:-------:|
| ≤ 20% | **1** | 极便宜 | 位于历史最低 1/5 区间，买入信号强烈 |
| 20% ~ 40% | **3** | 便宜 | 低于历史中位数，具备安全边际 |
| 40% ~ 60% | **5** | 中性 | 估值处于历史中位水平 |
| 60% ~ 80% | **7** | 偏贵 | 高于历史中位数，需谨慎观望 |
| > 80% | **9** | 极贵 | 位于历史最高 1/5 区间，泡沫风险 |

### 权重自动重分配

当某因子数据缺失时（如美股指数无法获取 A 股口径的 PB 数据），系统不会跳过该指数，而是将缺失因子的权重按比例分配至其余有效因子：

```
# 示例：红利型模板，股息率缺失
# 原始权重：股息率 40% + PE 35% + 价格位置 25%
# 重分配后：PE 58.3% (35/60) + 价格位置 41.7% (25/60)
```

## 数据源

| 数据源 | 数据类型 | 覆盖范围 | 优先级 |
|-------|---------|---------|:-----:|
| **理杏仁 Open API** | PE/PB/股息率 + 5年分位点 | 所有 A 股指数（中证、国证、策略指数等） | 估值数据首选 |
| **AkShare** | 历史行情（OHLCV） | A 股指数、美股指数 | 行情数据首选 |

**理杏仁 API 优势**：
- 一个 API 调用同时返回 PE/PB/股息率 + 分位点 + 最大/最小/20%/50%/80% 分位值
- 覆盖所有 A 股指数，不像 Tushare `index_dailybasic` 只覆盖 5 个宽基指数
- 支持批量查询（最多 100 个指数/次）、时间范围查询（日频数据）
- 分位点直接返回，无需自行从历史数据计算

**数据清洗规则：**

- OHLC 缺失值：用收盘价回填
- High < Low：自动互换（数据源异常）
- 成交量 NaN：置为 0
- 估值 0.0 哨兵值：转换为 None（标记为无数据）
- 负值 PE/PB：转换为 None（异常数据）
- 负值股息率：转换为 None（异常数据）

## 快速开始

### 安装

```bash
# 克隆项目
git clone https://github.com/your-username/index-score.git
cd index-score

# 创建虚拟环境
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate

# 安装（开发模式）
pip install -e ".[dev]"
```

### 配置

1. 配置理杏仁 API Token（估值数据源）：

```bash
# 复制环境变量模板
cp .env.example .env

# 编辑 .env，填入理杏仁 Token
# LIXINGER_TOKEN=your_token_here
```

2. 编辑 `config.yaml` 自定义指数列表、打分权重和 LLM 服务：

```yaml
# 添加新指数
indexes:
  - code: "000300"
    name: "沪深300"
    market: "CN"
    template: "balanced"  # 选择打分模板
    lixinger_code: "000300"  # 理杏仁指数代码

# 调整因子权重
scoring:
  templates:
    dividend:
      factors:
        - { field: "dividend_yield_percentile_5y", weight: 0.45 }  # 提高股息率权重
        - { field: "pe_percentile_5y", weight: 0.35 }
        - { field: "price_position_percentile_3y", weight: 0.20 }

# 配置 LLM（支持 DeepSeek / OpenAI / 通义千问）
llm:
  provider: "deepseek"
  model: "deepseek-chat"
  api_key_env: "INDEX_SCORE_LLM_API_KEY"
```

### 运行

```bash
# 设置环境变量（或在 .env 文件中配置）
export INDEX_SCORE_LLM_API_KEY="your-api-key"
export LIXINGER_TOKEN="your-lixinger-token"

# 启动终端界面
python main.py
```

### 交互操作

| 按键 | 功能 |
|:----:|------|
| `R` | 刷新数据（重新拉取所有指数行情和估值） |
| `Enter` | 查看指数详情（因子拆解 + LLM 解读） |
| `G` | 生成 Markdown 报告（保存到 `report/` 目录） |
| `Q` | 退出 |
