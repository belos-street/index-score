# 大盘指数量化打分 Agent 需求文档

## 1. 简介

### 1.1 项目定位

大盘指数量化打分 Agent（以下简称 Agent）是一款基于 Python 技术栈的命令行工具，专注于对主流大盘指数进行量化打分，以"1-9 分"为评分标准（分数越低代表指数估值越便宜，越具备长期投资价值），通过终端 UI 展示打分结果，并支持生成 Markdown 格式投资参考报告。

### 1.2 目标用户

长期价值投资者、指数基金投资者。适配低频长期投资场景，每日查询 1-2 次即可满足投资决策需求。

### 1.3 核心价值

- **简化决策**：量化模型替代人工计算，一键输出标准化打分结果
- **客观透明**：固定模型确保打分一致性，因子权重和规则可配置
- **轻量运行**：终端 UI 无冗余操作，Python 环境即可运行

## 2. 功能定义

### 2.1 核心功能

#### 2.1.1 指数量化打分

对预设指数列表，基于 3 因子加权模型自动计算 1-9 分估值分数。

**预设指数列表**（可在配置文件中增删）：

| 指数名称 | 指数代码 | 市场 |
|---------|---------|------|
| 中证红利 | 000922 | CN |
| 中证红利低波 | 930955 | CN |
| 标普中国红利低波50 | 930735 | CN |
| 国证价值100 | 399378 | CN |
| 国证自由现金流 | 980092 | CN |
| 纳指 | IXIC | US |
| 标普500 | SPX | US |

#### 2.1.2 终端 UI 展示

基于 Textual 的交互式终端界面，展示打分结果和因子明细。详见第 5 节。

#### 2.1.3 LLM 报告解读

集成 LangChain 框架，基于打分结果和因子数据生成自然语言投资解读。详见第 4.3 节。

#### 2.1.4 报告生成

手动触发，生成 Markdown 格式标准化报告，保存至 `report/` 目录。报告中的投资解读由 LLM 自动生成。详见第 6 节。

### 2.2 辅助功能

- **数据获取**：通过 AkShare（首选）/ Tushare（备选）拉取指数行情和估值数据，手动触发
- **异常处理**：API 请求失败自动重试、数据缺失自动调整权重、计算异常友好提示

## 4. 技术细节

### 4.1 数据获取

#### 4.1.1 触发方式

仅支持手动触发（通过终端 UI 的"刷新数据"按钮），不支持自动拉取和定时任务。

#### 4.1.2 行情数据字段

| 字段 | 类型 | 说明 |
|------|------|------|
| code | str | 指数代码 |
| name | str | 指数名称 |
| date | str | 日期 (YYYY-MM-DD) |
| open | float | 开盘价 |
| close | float | 收盘价 |
| high | float | 最高价 |
| low | float | 最低价 |
| volume | float | 成交量 |
| adj_close | float | 后复权价（用于计算价格位置） |

#### 4.1.3 估值数据字段

| 字段 | 类型 | 说明 |
|------|------|------|
| code | str | 指数代码 |
| date | str | 日期 |
| pe_ttm | float | PE(TTM) |
| pe_percentile_5y | float | PE 近 5 年分位 (0-100%) |
| pb_lf | float | PB(LF) |
| pb_percentile_5y | float | PB 近 5 年分位 (0-100%) |
| dividend_yield | float | 股息率 (%) |
| dividend_yield_percentile_5y | float | 股息率近 5 年分位 (0-100%) |

#### 4.1.4 数据口径

- 所有分位数据采用"从小到大排序"（分位值越低，代表当前指标越优）
- 估值分位采用近 5 年数据
- 价格位置分位采用近 3 年数据

#### 4.1.5 容错策略

- AkShare 请求失败：自动重试 3 次，每次间隔 3 秒
- 重试失败：自动切换至 Tushare 获取对应数据
- Tushare 也失败：终端提示"数据获取失败，请稍后重试"，终止该指数的数据拉取，不进行兜底填充
- 该指数标记为"数据不可用"，不参与本次打分

### 4.2 量化打分模型

#### 4.2.1 核心因子及权重

| 因子 | 权重 | 数据来源 | 统计周期 | 说明 |
|------|------|---------|---------|------|
| 股息率分位 | 40% | dividend_yield_percentile_5y | 近 5 年 | 分位越低 → 当前股息率越高 → 越便宜 |
| PE 估值分位 | 35% | pe_percentile_5y | 近 5 年 | 分位越低 → 当前估值越低 → 越便宜 |
| 价格位置分位 | 25% | 自行计算 | 近 3 年 | 辅助验证指标 |

权重可在配置文件中调整。

#### 4.2.2 价格位置计算

```
position = (当前后复权价 - 近3年最低后复权价) / (近3年后复权价 - 近3年最低后复权价) × 100%
```

#### 4.2.3 单因子打分规则

| 分位值/位置值 | 分数 | 标签 |
|-------------|------|------|
| ≤ 20% | 1 分 | 极便宜 |
| 20% < x ≤ 40% | 3 分 | 便宜 |
| 40% < x ≤ 60% | 5 分 | 中性 |
| 60% < x ≤ 80% | 7 分 | 偏贵 |
| > 80% | 9 分 | 极贵 |

打分区间可在配置文件中调整。

#### 4.2.4 总分计算

```
总分 = 股息率因子分 × 0.40 + PE因子分 × 0.35 + 价格位置因子分 × 0.25
```

结果保留 2 位小数。实际分数范围为 1.00-9.00。

#### 4.2.5 数据缺失处理

若某指数的某因子数据缺失（如股息率分位无法获取）：

1. 该因子权重按比例分配至其余因子
2. 示例：股息率缺失 → PE 权重调整为 35/(35+25) × 100% ≈ 58.3%，价格位置调整为 41.7%
3. 终端提示"该指数股息率数据缺失，已调整权重计算"

### 4.3 LangChain Agent

#### 4.3.1 架构设计

使用 LangChain 框架构建 Agent，将核心业务函数封装为 LangChain 工具（Tool），Agent 可根据任务需要自动选择调用对应工具。

```
┌─────────────────────────────────────┐
│           LangChain Agent           │
│  ┌─────┐ ┌─────┐ ┌─────┐ ┌──────┐  │
│  │打分 │ │对比 │ │配置 │ │报告  │  │
│  │查询 │ │分析 │ │建议 │ │生成  │  │
│  └─────┘ └─────┘ └─────┘ └──────┘  │
│         LangChain Tools             │
├─────────────────────────────────────┤
│         LLM (可切换)                │
│  OpenAI / DeepSeek / 通义千问       │
└─────────────────────────────────────┘
```

#### 4.3.2 工具封装

将以下函数封装为 LangChain Tool：

| 工具名称 | 功能 | 输入 | 输出 |
|---------|------|------|------|
| get_index_score | 查询单个或全部指数打分结果 | 指数代码（可选） | IndexScore 列表 |
| compare_indexes | 对比多个指数的打分和因子差异 | 指数代码列表（2-3个） | 对比分析文本 |
| get_allocation | 基于打分结果生成配置建议 | 总金额、策略偏好 | 配置比例建议 |
| generate_report | 生成 Markdown 报告 | 无 | 报告文件路径 |

#### 4.3.3 大模型支持

通过配置文件切换，支持所有兼容 OpenAI 接口的大模型：

| 提供商 | 模型示例 | base_url |
|--------|---------|----------|
| OpenAI | gpt-4o / gpt-4o-mini | https://api.openai.com |
| DeepSeek | deepseek-chat | https://api.deepseek.com |
| 通义千问 | qwen-plus | https://dashscope.aliyuncs.com/compatible-mode |

#### 4.3.4 报告解读生成

调用 LangChain Agent 自动生成投资报告中的解读部分：

- **输入**：当日全部指数打分结果 + 因子明细
- **输出**：每个指数的自然语言解读 + 报告摘要文本
- **触发**：用户在终端 UI 点击"生成报告"时自动触发

报告解读必须包含以下四部分：

1. **当前打分**：指数名称 + 总分 + 估值水平标签
2. **因子明细**：各因子的分位值和对应分数
3. **估值判断**：基于打分结果的整体估值水平描述
4. **投资建议**：长期投资视角的操作建议（持有/加仓/观望）

#### 4.3.5 容错策略

- LLM API 调用超时（30 秒）：返回兜底文本"LLM 解读生成超时，请稍后重试"
- LLM API 调用失败：返回兜底文本"LLM 解读生成失败，建议参考打分结果自行判断"
- 兜底文本不影响打分结果的展示

### 4.4 配置文件

配置文件位于项目根目录 `config.yaml`，格式如下：

```yaml
indexes:
  - code: "000922"
    name: "中证红利"
    market: "CN"
  - code: "930955"
    name: "中证红利低波"
    market: "CN"
  # ... 更多指数

scoring:
  dividend_yield_weight: 0.40
  pe_weight: 0.35
  price_position_weight: 0.25
  score_ranges:
    - { max_percentile: 20, score: 1 }
    - { max_percentile: 40, score: 3 }
    - { max_percentile: 60, score: 5 }
    - { max_percentile: 80, score: 7 }
    - { max_percentile: 100, score: 9 }

llm:
  provider: "deepseek"
  model: "deepseek-chat"
  api_key: ""           # 留空时从环境变量 LLM_API_KEY 读取
  base_url: "https://api.deepseek.com"
  timeout: 30

report:
  show_detail: true          # 是否展示单个指数详细分析
  sort_by: "score"           # 排序方式：score（按打分）/ name（按名称）
```

### 4.6 日志策略

| 级别 | 内容 | 存储 |
|------|------|------|
| INFO | 数据拉取成功、打分计算完成、报告生成成功 | 终端输出 |

### 4.7 后续迭代功能（V2）

以下功能基于 LangChain Agent 工具扩展，不在 V1 范围内，但架构已在 4.3.2 工具封装中预留。

#### 4.7.1 自然语言问答

用户可在终端详情面板输入自然语言问题，Agent 基于打分数据推理回答。

**示例**：
- "为什么中证红利低波比中证红利便宜？"
- "纳指现在 7 分，历史上处于什么水平？"
- "红利类指数整体估值怎么样？"

**交互入口**：详情面板底部增加输入框，Enter 提交问题。

#### 4.7.2 仓位配置建议

基于打分结果，为用户生成指数间的配置比例建议。

**输入**：总金额、策略偏好（等权/估值加权）
**输出**：各指数配置金额和比例

**示例**：
- 输入：10 万，估值加权
- 输出：中证红利低波 3 万（30%）、标普红利低波 2.5 万（25%）...

**交互入口**：首页增加【A】配置建议按钮。

#### 4.7.3 开发计划

V1 完成核心打分 + Textual UI + 报告生成后，优先实现 4.7.2（仓位配置建议），再实现 4.7.1（自然语言问答）。

### 5.2 标题区

居中显示"大盘指数量化打分 Agent"，右侧显示：

- 数据来源：AkShare / Tushare
- 更新时间：YYYY-MM-DD HH:MM

### 5.3 核心表格区

| 指数名称 | 指数代码 | 当前打分 | 股息率因子分 | PE因子分 | 价格位置因子分 |
|---------|---------|---------|------------|---------|-------------|
| 标普中国红利低波50 | 930735 | 2.78 | 1 | 1 | 3 |

- 支持滚动查看

底部显示可用操作：

- **【R】刷新数据** — 重新拉取所有指数数据并计算打分

V2 预留：
- **【A】配置建议** — 基于打分结果生成仓位配置建议
- 详情面板内支持自然语言问答输入

### 5.5 指数详情面板

选中某个指数后，展示详细信息：

- **指数基础信息**：名称、代码、当前日期
- **因子明细**：
  - 股息率分位（近5年）：22% → 打分 3 分（便宜）
  - PE 估值分位（近5年）：18% → 打分 1 分（极便宜）
  - 价格位置分位（近3年）：30% → 打分 3 分（处于历史低位区间）
- **LLM 解读**（灰色背景突出）：自然语言投资解读文本
- **操作提示**：【B】返回首页

### 5.6 操作反馈

- 刷新数据：显示"正在更新数据，请稍候..." → "数据更新成功，共更新 N 个指数"
- 生成报告：显示"正在生成报告..." → "报告已保存至 ./report/指数打分报告_YYYYMMDD.md"
- 数据获取失败：显示"指数 XXX 数据获取失败，已跳过"

### 5.7 初始状态

首次启动时，表格为空，自动触发一次数据刷新。刷新完成后展示打分结果。

## 6. 报告设计

### 6.1 触发方式

仅支持通过终端 UI 的"生成报告"按钮手动触发。

### 6.2 报告格式

Markdown，保存至 `report/指数打分报告_YYYYMMDD.md`。

### 6.3 报告结构

```markdown
# 大盘指数量化打分报告（YYYY-MM-DD）

今日共覆盖 N 个主流大盘指数，平均打分为 X.XX 分，整体处于 [估值水平]。
其中，[最低指数] 打分最低（X.XX 分，[标签]），[最高指数] 打分最高（X.XX 分，[标签]）。

## 二、指数打分明细

| 指数名称 | 指数代码 | 当前打分 | 股息率因子分 | PE因子分 | 价格位置因子分 |
|---------|---------|---------|------------|---------|-------------|
| ... | ... | ... | ... | ... | ... |

## 三、单个指数详细分析

### 3.1 [指数名称]（代码：XXX，打分：X.XX 分）

- **因子明细**：
  - 股息率分位（近5年）：XX% → 打分 X 分（[标签]）
  - PE估值分位（近5年）：XX% → 打分 X 分（[标签]）
  - 价格位置分位（近3年）：XX% → 打分 X 分（[标签]）
- **投资解读**：[LLM 生成的自然语言解读]

### 3.2 ... (每个指数一个子章节)

## 四、数据来源及计算逻辑

- 数据来源：AkShare / Tushare，拉取日期 YYYY-MM-DD
- 量化模型：3 因子加权平均（股息率 40% + PE 35% + 价格位置 25%）
- 分位统计周期：估值分位近 5 年，价格位置近 3 年

---
报告生成时间：YYYY-MM-DD HH:MM
```

## 7. 异常处理汇总

| 异常场景 | 处理策略 | 用户提示 |
|---------|---------|---------|
| AkShare 请求失败 | 重试 3 次（间隔 3 秒）→ 切换 Tushare | "正在重试..." |
| AkShare + Tushare 均失败 | 终止该指数数据拉取 | "该指数数据获取失败，请稍后重试" |
| 某因子数据缺失 | 该因子权重按比例分配至其余因子 | "该指数 XX 数据缺失，已调整权重计算" |
| 数据格式错误 | 记录日志，该指数标记为不可用 | "该指数数据异常，已跳过" |
| 除零异常 | 返回兜底值，记录日志 | "计算异常，请刷新数据重试" |
| LLM API 超时 | 返回兜底解读文本 | "LLM 解读生成超时" |
| LLM API 失败 | 返回兜底解读文本 | "LLM 解读生成失败" |
| 配置文件缺失 | 使用默认配置 | "未找到 config.yaml，使用默认配置" |
| 配置文件格式错误 | 启动失败 | "config.yaml 格式错误：[具体错误]"
